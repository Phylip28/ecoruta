MVP
Servicio de Autenticación JWT Multi-Rol
Registro, login y refresh de tokens. Decoradores de permisos por rol (ciudadano, reciclador, admin). Password hasheado con bcrypt.

Libreríapython-jose · passlib[bcrypt] · fastapi-security

BE-02
MVP HU-1 HU-2
API de Reportes y Solicitudes (CRUD)
Endpoints para crear, listar y actualizar reportes. Diferencia entre tipo EMERGENCIA y SOLICITUD_RECOLECCION. Almacena imagen, coordenadas, tipo de material y estado.
EndpointsPOST /reportes · GET /reportes · PATCH /reportes/{id}/estado
StorageImágenes en volumen Docker, ruta relativa en PostgreSQL
Prioridad● Crítico
BE-03
MVP HU-3
Gestor de Estados de Recolección
Máquina de estados que valida transiciones legales (PENDIENTE→EN_CAMINO→COMPLETADO). Al completar, dispara el cálculo de kg desviados y actualiza métricas del reciclador.
EndpointsPATCH /recolecciones/{id}/estado
PatrónState Machine + Event Emitter interno
DoDCriterio de Aceptación #3 ✅
Prioridad● Crítico
BE-04
MVP HU-3
Motor de Optimización de Rutas
Recibe ubicación del reciclador + lista de solicitudes pendientes y devuelve la secuencia óptima usando Nearest Neighbor (TSP heurístico). Filtrable por radio en km.
EndpointsPOST /rutas/optimizar → lista ordenada de coordenadas
AlgoritmoNearest Neighbor greedy con scipy.spatial (open source)
Input{lat, lng, radio_km} → array de puntos ordenados
Prioridad● Crítico
BE-05
MVP HU-4
Cálculo de Huella de Carbono e Indicadores
Algoritmo que convierte kg recolectados por tipo de material en CO₂ equivalente evitado. Factores de emisión por material basados en datos EPAs. Expone endpoint para el dashboard.
EndpointsGET /metricas/impacto → kg_total, co2_evitado_kg, arboles_equiv
FactoresPlástico: 1.9 kg CO₂/kg · Cartón: 0.9 · Vidrio: 0.3 · Metal: 4.0
DoDCriterio de Aceptación #5 ✅
Prioridad● Crítico

Arquitectura e Infraestructura
Patrones, stack de datos y configuración Docker
3 features
ARCH-01
ARCHMVP
Docker Compose Multi-Servicio
Orquesta FastAPI + PostgreSQL/PostGIS + (opcional) Redis en un solo comando. Volúmenes para persistencia de imágenes y DB. Variables de entorno con .env para configuración.
Serviciosapi (FastAPI) · db (postgis/postgis:15) · redis (opcional)
RelevanciaGarantiza demo funcional en cualquier máquina el día del hackathon
Prioridad● Crítico
ARCH-02
ARCH
Arquitectura Layered (Controllers / Services / Repositories)
Separación clara en 3 capas en el backend: routers FastAPI, capa de servicios con lógica de negocio y capa de repositorios con acceso a DB. Facilita testing y escalabilidad post-hackathon.
Estructuraapp/routers/ · app/services/ · app/repositories/ · app/models/
PatrónRepository Pattern + Dependency Injection de FastAPI
Prioridad● Alta
ARCH-03
ARCH
Modelo de Datos Geoespacial (Schema DB)
Tablas principales con tipos geoespaciales: usuarios, reportes (punto geography), recolecciones, materiales, métricas_reciclador, audit_log. Índices espaciales en columnas de coordenadas.
Tablasusers · reports · collections · materials · metrics · audit_log
ÍndicesGIST index en columna location (geography type)
Prioridad● Alta
